MARKDOWN BODY (after the closing triple-dash):

Section "## Overview":
  Write: "llm-wiki is a personal knowledge engine that turns questions into wiki articles. Every question either retrieves an existing wiki answer or searches the web and synthesizes new articles into your Obsidian vault."
  Write: "**stdout** contains machine-readable output (answers, article titles, JSON)."
  Write: "**stderr** contains progress and status messages. Ignore stderr when parsing output."

Section "## Commands":

  Subsection "### Ask a question":
    Command: `wiki ask "<question>"`
    Behavior bullets:
    - Checks the local wiki first. If relevant articles exist, outputs an answer to stdout.
    - If the wiki lacks coverage, searches the web, synthesizes articles, and outputs article title(s) to stdout (one per line).
    Flags:
    - `--web` -- Skip wiki check; always search the web and synthesize new articles.
    - `--refresh` -- Re-fetch web sources for existing articles older than the configured freshness_days (default: 30 days). If the article is fresh, answers from wiki. If no matching article exists, falls through to normal web search.
    stdout format:
    - Wiki path: plain text answer
    - Web path: article title(s), one per line

  Subsection "### Search the wiki":
    Command: `wiki search "<query>"`
    stdout: JSON array of matching articles. Each entry has `slug`, `title`, `summary`, and `score` fields.
    Example: `wiki search "attention mechanism"`

  Subsection "### List all articles":
    Command: `wiki list`
    stdout: JSON array of all articles with `slug`, `title`, `categories`, and `updated_at`.

  Subsection "### Ingest a URL":
    Command: `wiki ingest <url>`
    Description: Fetches the URL, extracts content, and synthesizes it into the wiki. Supports web pages, PDFs, and arxiv papers.
    stdout: article title on success.
    stderr: progress messages.

Section "## Parsing Output":
  - Always read stdout for machine-readable data.
  - Never parse stderr -- it is for human display only.
  - Exit code 0 = success. Non-zero = error (check stderr for message).

Section "## Installation":
  1. Install globally: `npm install -g llm-wiki`
  2. Verify: `wiki --help`
  3. Place this skill in `~/.openclaw/skills/llm-wiki/SKILL.md` or keep it in your workspace `skills/` directory.

The resulting scripts block should be:
```json
"scripts": {
  "build": "tsc",
  "prepare": "npm run build",
  "prepublishOnly": "npm run build",
  "dev": "tsx src/index.ts",
  "test": "vitest run",
  "test:watch": "vitest"
}
```

2. Add a `"files"` field to package.json at the top level (after `"bin"`):
```json
"files": [
  "dist/",
  "skills/",
```

This limits the published package to compiled output, the skill manifest, and package.json. It prevents dev artifacts, tests, .planning/, and source TypeScript from shipping.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SKILL.md metadata -> OpenClaw loader | Static YAML+JSON; no runtime user input |
| npm registry -> user machine | Published package must include dist/ with compiled JS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | SKILL.md metadata injection | accept | metadata is a static JSON string authored by developer -- no user input accepted at runtime; OpenClaw parses it as read-only skill manifest |
| T-06-05 | Information Disclosure | stdout in subprocess | accept | stdout already contains only machine-readable data (article titles, search results, answers) -- no secrets, API keys, or credentials exposed |
| T-06-06 | Denial of Service | npm prepare script | accept | prepare runs only during `npm install -g .` (local); limited to `tsc` compilation; cannot be triggered by external callers |
</threat_model>
