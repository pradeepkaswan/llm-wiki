    const PLAN_TEMPERATURE = 0.2;
    const GENERATE_TEMPERATURE = 0.3;

    **`synthesize()` function flow:**

    ```typescript
    export async function synthesize(
      rawDir: string,
      store: WikiStore,
    ): Promise<SynthesisResult> {
    ```

    **Step 1: Load source envelopes from manifest.**
    - Read `manifest.json` from `rawDir`
    - For each non-excluded ManifestEntry, read the source JSON file
    - Parse into `RawSourceEnvelope[]`
    - If no non-excluded envelopes, throw error: "No usable source envelopes found"

    **Step 2: Load existing articles for backlink constraint and dedup.**
    - Call `store.listArticles()` to get all existing articles
    - Extract slug list for wikilink constraint

    **Step 3: Plan articles (per D-08, D-09).**
    - Build the planning input: `{ question: manifest.query, envelopes, existingArticles }`
    - Call `generateText(buildPlanPrompt(input), { system: SYSTEM_PROMPT, temperature: PLAN_TEMPERATURE, maxOutputTokens: 1024 })`
    - Parse with `parsePlanOutput(raw, envelopes.length)`
    - Deduplicate planned titles within the batch: if two plans produce the same slug (via `store.slugify()`), keep only the first one (per RESEARCH Pitfall 3)

    **Step 4: Generate each article (per D-08 Step 2).**

    For each `ArticlePlan` in the plan:

    a. **Dedup check** — call `findExistingArticle(plan.title, store, existingArticles)`

    b. **Select relevant sources** — `plan.sourceIndices.map(i => envelopes[i]).filter(Boolean)`. If sourceIndices is empty or all invalid, use all envelopes as fallback.

    c. **Build known slugs set** — union of existing article slugs + slugs of articles already generated in this batch (per D-10)

    d. **Generate** — If dedup found an existing article, use `buildUpdatePrompt(existing, relevantSources, question, knownSlugs)`. Otherwise use `buildGeneratePrompt(question, plan, relevantSources, knownSlugs)`. Call with `{ system: SYSTEM_PROMPT, temperature: GENERATE_TEMPERATURE, maxOutputTokens: MAX_OUTPUT_TOKENS }`.

    e. **Parse** — Call `parseArticleOutput(raw)`. If null (parse failure per D-03), retry ONCE with a stricter note appended to the prompt: "IMPORTANT: You MUST follow the exact format. Start with TITLE: on the first line, then SUMMARY:, then CATEGORIES:, then BODY:". If still null after retry, throw error: `Synthesis failed: could not parse LLM output for "${plan.title}" after retry`.

    f. **Build Article** — If updating existing, call `buildUpdatedArticle(existing, parsed, knownSlugsSet)`. Otherwise `buildNewArticle(parsed, knownSlugsSet)`.

    g. **Save** — Call `store.saveArticle(article)`. This validates frontmatter (SYNTH-06) and rebuilds index. Log to stderr: `[SAVED] articles/{slug}.md`

    h. **Track** — Push article to results array. Add slug to knownSlugs set for next iteration.

    **Step 5: Return SynthesisResult.**
    - `articles`: all saved articles
    - `updatedSlugs`: slugs where dedup found an existing article

    **Error handling:**
    - Per RESEARCH Pitfall 6: only retry on parse failure (parseArticleOutput returns null). API errors from generateText propagate immediately.
    - Per RESEARCH Pitfall 4: source URLs in article come from SourceRef[] in parsed output (not raw envelopes) — only cited sources.
