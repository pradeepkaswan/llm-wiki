    Add a `GenerateOptions` interface and update the `generateText()` function signature:

    ```typescript
    export interface GenerateOptions {
      system?: string;
      temperature?: number;
      maxOutputTokens?: number;
    }

    export async function generateText(
      prompt: string,
      options: GenerateOptions = {}
    ): Promise<string> {
      const config = await loadConfig();
      const model = createProvider(config);
      const { text } = await sdkGenerateText({
        model,
        prompt,
        system: options.system,
        temperature: options.temperature,
        maxOutputTokens: options.maxOutputTokens,
      });
      return text;
    }
    ```

    This is backward-compatible: existing callers pass only `prompt` and get identical behavior. The Vercel AI SDK `CallSettings` type already supports `system`, `temperature`, and `maxOutputTokens` (verified in node_modules/ai/dist/index.d.ts). Use `maxOutputTokens` NOT `maxTokens` (deprecated in AI SDK v4+).

    ```typescript
    import type { Article } from '../types/article.js';
    import type { RawSourceEnvelope } from '../types/ingestion.js';

    /** Input to the synthesis pipeline */
    export interface SynthesisInput {
      question: string;
      envelopes: RawSourceEnvelope[];  // only non-excluded envelopes
      existingArticles: Article[];      // all articles currently in wiki
    }

    /** Output from the planning step — one entry per planned article */
    export interface ArticlePlan {
      title: string;
      scope: string;                    // brief description of what this article covers
      sourceIndices: number[];          // indices into SynthesisInput.envelopes
    }

    /** Parsed output from a single article generation LLM call */
    export interface ParsedArticle {
      title: string;
      summary: string;
      categories: string[];
      body: string;                     // markdown body WITH inline [N] citations
      sourceRefs: SourceRef[];          // the ## Sources entries
    }

    /** A single source reference from the ## Sources section */
    export interface SourceRef {
      index: number;                    // the [N] number
      title: string;
      url: string;
    }

    /** Final result of the synthesis pipeline */
    export interface SynthesisResult {
      articles: Article[];              // saved articles
      updatedSlugs: string[];           // slugs that were updates (not new)
    }
    ```

    **1. Create src/synthesis/prompt-builder.ts (per D-02, D-06, D-08, D-09, D-13):**

    Four pure functions that construct LLM prompt strings. Each source envelope's markdown content is truncated to 3000 characters maximum (per RESEARCH Pitfall 2 — prevents token limit exceeded with many sources).

    `buildPlanPrompt(input: SynthesisInput): string`
    - System context: "You are a wiki planning assistant..."
    - Include the user's question
    - List each non-excluded source: index, title, URL, first 500 chars of content
    - List existing article titles (for awareness of what already exists)
    - Instruct the LLM to output in this exact format:
      ```
      ARTICLE_COUNT: N
      ARTICLE_1_TITLE: ...
      ARTICLE_1_SCOPE: ...
      ARTICLE_1_SOURCES: 0,2,3
      ARTICLE_2_TITLE: ...
      ARTICLE_2_SCOPE: ...
      ARTICLE_2_SOURCES: 1,4
      ```
    - Single-topic questions produce ARTICLE_COUNT: 1 with all source indices
    - Broad questions produce ARTICLE_COUNT: 2+ with partitioned sources

    `buildGeneratePrompt(question: string, plan: ArticlePlan, sources: RawSourceEnvelope[], knownSlugs: string[]): string`
    - Per D-02: include full markdown of each relevant source (truncated to 3000 chars)
    - Per D-04: instruct inline `[N]` citations in body, `## Sources` at bottom
    - Per D-06: include known article slugs with instruction to only use `[[slug]]` for articles in the list
    - Instruct the LLM to output in this exact format:
      ```
      TITLE: ...
      SUMMARY: ...
      CATEGORIES: cat1, cat2
      BODY:
      ## Section 1
      ...
      ## Sources
      1. [Title](url)
      ```

    `buildUpdatePrompt(existingArticle: Article, newSources: RawSourceEnvelope[], question: string, knownSlugs: string[]): string`
    - Per D-13: include existing article body, new source material, instructions to incorporate new information while preserving existing structure
    - Same output format as buildGeneratePrompt

    `buildTiebreakPrompt(plannedTitle: string, existingSummary: string): string`
    - Per D-11 tier 3: ask LLM if the planned article should update the existing one or be a new article
    - Instruct to respond with exactly `UPDATE` or `NEW`

    Each prompt must include explicit format instructions and warn the LLM NOT to wrap output in markdown code fences.

    **2. Create src/synthesis/output-parser.ts (per D-03, RESEARCH Pattern 5):**

    Three pure functions that parse LLM text responses. Parse defensively per RESEARCH Pitfall 1: strip leading/trailing markdown code fences, scan line-by-line for delimiters rather than splitting on exact positions.

    `parsePlanOutput(raw: string, sourceCount: number): ArticlePlan[]`
    - Strip markdown code fences if present (``` at start/end)
    - Find `ARTICLE_COUNT:` line, parse integer N
    - For each article 1..N, find `ARTICLE_{i}_TITLE:`, `ARTICLE_{i}_SCOPE:`, `ARTICLE_{i}_SOURCES:` lines
    - Parse source indices from comma-separated string, validate they are in range [0, sourceCount-1]
    - Fallback: if parsing fails (no ARTICLE_COUNT found or parse error), return a single ArticlePlan with title derived from the raw text (first non-empty line) and all source indices [0..sourceCount-1]

    `parseArticleOutput(raw: string): ParsedArticle | null`
    - Strip markdown code fences if present
    - Find `TITLE:` line — extract title (trim whitespace)
    - Find `SUMMARY:` line — extract summary (trim whitespace)
    - Find `CATEGORIES:` line — split on comma, trim each, default to `['Uncategorized']` if missing
    - Find `BODY:` line — everything after it until end of string is the body
    - Parse `## Sources` section from body: extract numbered entries `N. [title](url)` into SourceRef[]
    - Return null if TITLE or BODY markers are not found (completely unparseable)

    `parseTiebreakDecision(raw: string): 'update' | 'new'`
    - Trim whitespace, convert to uppercase
    - If contains 'UPDATE', return 'update'
    - Otherwise return 'new' (safe default: create new rather than accidentally overwriting)
