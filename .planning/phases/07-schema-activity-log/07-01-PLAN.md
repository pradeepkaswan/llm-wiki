## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | appendLog description field | accept | Descriptions are constructed by production code, not user input. Single-line format prevents newline injection. Single-user personal tool — no untrusted actors. |
| T-07-02 | Tampering | schema.md content via updateSchema | accept | Schema content is either bootstrapped from code constants or deterministically appended. No external input path. writeFileAtomic prevents partial-write corruption. |
| T-07-03 | Information Disclosure | log.md exposes operation history | accept | log.md is a local file in the user's personal vault. No network exposure. This is the intended behavior per LOG-01. |
| T-07-04 | Denial of Service | Unbounded log.md growth | accept | Append-only log grows linearly with wiki operations. A personal wiki generates ~100s of entries over months. No mitigation needed at this scale. |
