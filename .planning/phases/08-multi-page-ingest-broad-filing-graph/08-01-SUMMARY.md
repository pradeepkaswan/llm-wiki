## What Was Built

### `src/synthesis/see-also.ts`

Shared `upsertSeeAlsoEntry(body, entry)` utility. Handles all four scenarios:
- No `## See Also` section + no `## Sources`: appends at end of body
- No `## See Also` section + has `## Sources`: inserts before Sources (canonical order)
- Existing `## See Also` section: inserts entry into the list
- Entry slug already in See Also: returns body unchanged (idempotent)

### `src/synthesis/ripple.ts`

`rippleUpdates(primaryArticles, store, schema)` — propagates cross-references to related articles:
1. Loads all existing articles from `store.listArticles()`
2. Builds a transient BM25 index via `buildIndex()`
3. Queries each primary's `title + summary`, takes top-10 per primary
4. Excludes primary slugs, filters below BM25 score 3.0 (same as `BM25_DEDUP_THRESHOLD`)
5. Deduplicates target slugs across all primaries
6. Makes a **single batched LLM call** (D-04) with all primary and target summaries
7. Parses response: strips markdown code fences, `JSON.parse()` — returns empty on failure (T-08-01)
8. Applies `upsertSeeAlsoEntry` to each target, saves via `store.saveArticle(article, 'update')` sequentially

### `src/synthesis/backlink-enforcer.ts`

`enforceBacklinks(article, store)` — adds reciprocal backlinks for all wikilinks in an article:
1. Extracts `[[slug]]` and `[[slug|display text]]` via `WIKILINK_RE`
2. Deduplicates slugs, removes article's own slug (no self-references)
3. For each target slug: reads via `store.getArticle()`, builds backlink entry, calls `upsertSeeAlsoEntry()`
4. Only saves if body changed (idempotency guard)
5. Returns list of actually-updated slugs
6. Processes sequentially to prevent index rebuild races

1. **`ripple.ts` calls `store.appendLog` after each save** — the plan said "Log entries auto-generated by saveArticle's appendLog", but `saveArticle` already calls `appendLog` internally. 2. **`upsertSeeAlsoEntry` uses `\n(##\s)` lookahead for next section detection** — the regex uses a newline-prefixed match to avoid false positives in body content that might start with `## `.
