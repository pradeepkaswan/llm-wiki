    **Implement 5 check functions (all private, called from runLint):**

    1. **checkOrphans(articles)** per D-03:
       - Define WIKILINK_RE locally: `const WIKILINK_RE = /\[\[([^\]|]+)(?:\|[^\]]+)?\]\]/g;`
       - Build reverse-link map: `Map<slug, Set<referencingSlug>>`
       - Initialize all slugs with empty Set
       - For each article body, create FRESH regex `new RegExp(WIKILINK_RE.source, 'g')`, exec loop to extract targets
       - Articles with empty inbound set → orphan finding
       - Severity: 'warning'
       - suggestedFix: `"Add backlink from a related article to [[${slug}]]"`

    2. **checkStale(articles, config)** per D-04:
       - Import `isArticleStale` from `../commands/ask.js`
       - For each article, call `isArticleStale(article, config.freshness_days ?? 30)`
       - Stale articles → finding with severity 'warning'
       - suggestedFix: `"Re-fetch by running: wiki ask \"${article.frontmatter.title}\" --refresh"`

    3. **checkMissingConcepts(articles)** per D-05:
       - Build known slugs set from articles array
       - Extract all wikilinks from all article bodies (fresh regex each time)
       - Any wikilink target not in known slugs → missing-concept finding
       - Severity: 'info'
       - suggestedFix: `"Create article for [[${slug}]] by running: wiki ask \"${displayTitle}\""`
       - Convert slug to display title: `slug.split('-').map(w => w.charAt(0).toUpperCase() + w.slice(1)).join(' ')` per RESEARCH Pitfall 4

    4. **checkMissingCrossRefs(articles)** per D-06:
       - Import `buildIndex`, `search` from `../search/search-index.js`
       - Guard: if articles.length < 2, return empty
       - Build BM25 index from articles
       - For each article, search with `article.frontmatter.title`
       - Filter results: exclude self (slug !== article.slug), score >= CROSS_REF_THRESHOLD (5.0), not already linked in body
       - Extract already-linked slugs via WIKILINK_RE from article body
       - Also check See Also section for existing links
       - Missing high-scoring matches → finding with severity 'info'
       - suggestedFix: `"Add cross-reference: [[${article.slug}]] ↔ [[${match.slug}]]"`
       - Deduplicate: if A→B already flagged, don't flag B→A

    5. **checkContradictions(articles)** per D-07:
       - Guard: if articles.length < 2, return empty array
       - Import `generateText` from `../llm/adapter.js`
       - Build article summary list: `- "Title" (slug): summary` for each article
       - Chunk if > 50 articles (batch in groups of 25)
       - Prompt: Ask LLM to identify contradictions. Return JSON array: `[{ "slugA": "...", "slugB": "...", "conflict": "..." }]`
       - Parse response: strip code fences, JSON.parse in try/catch
       - On parse failure: `process.stderr.write('[lint] ...\n')`, return empty array
       - On non-array: return empty array
       - Map each result to LintFinding with severity 'error'
       - suggestedFix: `"Review contradiction: ${conflict}"`

    - Build counts: iterate findings, count per category
    - Build healthScore: count articles that appear in no finding's `affected` array, divide by total articles, multiply by 100, round to 1 decimal

    **Command structure (follow file.ts pattern):**
    ```typescript
    import { Command } from 'commander';
    import { loadConfig } from '../config/config.js';
    import { WikiStore } from '../store/wiki-store.js';
    import { runLint } from '../lint/linter.js';
    import type { LintCategory } from '../lint/linter.js';

    const VALID_CATEGORIES: LintCategory[] = [
      'orphan', 'stale', 'missing-concept', 'missing-cross-ref', 'contradiction'
    ];

    export const lintCommand = new Command('lint')
      .description('Scan the wiki for health issues — orphans, stale articles, missing concepts, contradictions')
      .option('--category <type>', 'filter to a specific check type')
      .action(async (options: { category?: string }) => {
        try {
          const config = await loadConfig();
          const store = new WikiStore(config.vault_path);

          // Validate --category flag (V5 Input Validation — ASVS L1)
          let categories: LintCategory[] | undefined;
          if (options.category) {
            if (!VALID_CATEGORIES.includes(options.category as LintCategory)) {
              process.stderr.write(
                `Error: Invalid category "${options.category}". Valid: ${VALID_CATEGORIES.join(', ')}\n`
              );
              process.exit(1);
            }
            categories = [options.category as LintCategory];
          }

          // Read schema for LLM calls per D-18
          const schema = await store.readSchema();

          // Load all articles
          const articles = await store.listArticles();

          if (articles.length === 0) {
            process.stderr.write('No articles found in the wiki.\n');
            process.stdout.write(JSON.stringify({ findings: [], counts: {}, healthScore: 100, articleCount: 0 }) + '\n');
            return;
          }

          process.stderr.write(`Scanning ${articles.length} article(s)...\n`);

          // Run lint — passing config (contains freshness_days)
          const report = await runLint(articles, config, { categories });

          // D-09: JSON to stdout
          process.stdout.write(JSON.stringify(report) + '\n');

          // D-09: Human summary to stderr
          process.stderr.write('\n--- Wiki Health Report ---\n');
          process.stderr.write(`Health Score: ${report.healthScore}%\n`);
          process.stderr.write(`Total findings: ${report.findings.length}\n`);
          for (const [cat, count] of Object.entries(report.counts)) {
            if (count > 0) {
              process.stderr.write(`  ${cat}: ${count}\n`);
            }
          }
          if (report.findings.length === 0) {
            process.stderr.write('No issues found — wiki is healthy!\n');
          }

          // Log lint run to log.md per D-13
          await store.appendLog('lint', `Lint scan: ${report.findings.length} finding(s), health ${report.healthScore}%`);
        } catch (err: unknown) {
          const message = err instanceof Error ? err.message : String(err);
          process.stderr.write(`Error: ${message}\n`);
          process.exit(1);
        }
      });
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI --category flag → linter | User-supplied string must match known enum |
| LLM response → contradiction parser | Untrusted JSON from LLM must be validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | --category flag | mitigate | Validate against VALID_CATEGORIES array; reject unknown values with clear error message and process.exit(1) |
| T-09-02 | Information Disclosure | LLM contradiction prompt | accept | Article summaries sent to LLM are user's own wiki content; no PII beyond what user authored. Same trust boundary as existing synthesis calls. |
| T-09-03 | Denial of Service | Large wiki → LLM context overflow | mitigate | Chunk articles > 50 into batches of 25 for contradiction detection prompt. Guard articles.length < 2 to skip unnecessary LLM calls. |
| T-09-04 | Spoofing | LLM response parse | mitigate | Strip code fences, JSON.parse in try/catch, validate array structure. On failure, return empty array — never throw. Pattern from ripple.ts. |
</threat_model>
